# JSR-000004 ECperf基准规范


#### 规范描述

	用于XML解析的Java TM API（JAXP）使开发人员可以通过行业标准的SAX和DOM API在其应用程序中轻松使用XML解析器。

#### 规范最新阶段 

最终发行	下载页面	开始：2000-03-21	



#### 规范负责人

Oracle的Rajiv Mordani

#### 要求

```
A.背景
预期的规范将满足对支持XML 1.0的一整套与实现无关的可移植API的需求。XML规范可从以下网站在线获得：http : //www.w3.org/TR/REC-xml。

XML是独立于平台的数据表示形式，可以将其视为SGML的简化的Web感知版本。它是新一代Web技术的基础。如今，它已在Web应用程序服务器中用作动态内容生成系统的一部分，并作为消息传递系统（例如，用于企业对企业的Web商务和工作流）的一部分，以多种编程语言编写而成的系统组件。

用于XML的Java TM API的现有规范不能满足完整应用程序的全部要求（有关更多信息，请参见下文）。简而言之，公认的可移植XML API（SAX和DOM）在基本功能（例如验证，从输入文档构造DOM树，写出格式正确的XML以及使用XML名称空间）方面具有可移植性限制。

B.范围和内容
我们建议开发一组模块化的库API，一个100％纯Java TM 参考实现（RI）和一个兼容性测试套件（CTS），以至少解决以下问题。

它基于JDK 1.1 API集针对Java平台的桌面版和企业版。这些API也可以轻松地支持Personal Java平台。

公认的是，目前XML领域正在开展许多工作。该提案将提供一组核心功能，这些功能将构成基于XML的全功能应用程序的构建块。

我们建议该核心应包括：

（1）基于事件的XML解析。
我们期望该API将基于“ SAX 1.0” API，为此目的，该API已被广泛接受。可能需要扩展，以暴露那些API并未公开但核心功能集中其他地方必需的信息。

对于许多不能接受生成XML数据的内存表示形式的开销的应用程序，基于事件的解析是绝对必要的。

该程序包的名称是“ org.xml.sax”，还有“ org.xml.sax.helpers”。扩展可能不在这些软件包中。

（2）XML数据的可选基本内存表示形式。

我们希望该API将基于W3C“ DOM Level 1 Core” API。我们知道需要对该程序包进行扩展，因为无法获取实现或根据给定的XML源文档填充该实现。同样，没有定义的方法可以扩展“核心”实现以支持其他功能集，例如“ HTML DOM”或基于ID的节点访问（XSL和XLink支持所需）。

该程序包的名称是“ org.w3c.dom”，但扩展名不会出现在该程序包中，除非它们被W3C所采用。

（3）用于打印XML数据的基本API。

XML的应用程序涉及读取其他来源提供的XML或编写XML文本，以便其他应用程序可以处理它。需要一些API来支持编写XML文本，如果不是“有效”的话，至少要“格式正确”。

（4）支持XML命名空间

XML名称空间是一种约定，用于将URI与元素和属性相关联，以创建可扩展的命名框架来代替原始的平面名称空间。需要API来支持根据名称空间访问此类XML节点，否则需要利用此结构化工具。

我们预计，外部组织尚未定义的扩展将位于

javax.xml
包（或子包）。
C.含义
该技术没有直接的安全隐患。但是，它可以用于对安全敏感的上下文中，例如Web Commerce消息。

XML技术针对国际化系统。它是根据Unicode字符集定义的，支持多种（和可扩展的）字符编码，并且直接支持在同一XML文档中以多种语言表示文本。例如，在基于Web的系统中XML的普遍使用要求在同一Java虚拟机*中同时支持多种语言（例如，一个客户端使用英语，而另一客户端使用日语的情况）。

在本地化方面，此XML技术级别的参考实现将要求诊断消息和文档可翻译为本地语言。

不提供此JSR中概述的规范的风险是，核心XML API中将存在碎片。同样，不同实现之间的兼容性也将有很大的不同。尽管需要进行XML一致性工作（如下所述），但提供RI或CTS并没有特别的困难。

D.现有规格
W3C DOM工作组已经制定了“ Level 1 Core”规范，并且正在开发增强的“ Level 2 Core”。这是在OMG-IDL中指定的，并具有与Java和JavaScript的自定义语言绑定。

David Megginson，协调来自“ XML-DEV”邮件列表成员的输入，已经产生了SAX 1.0规范。该引用是用Java指定的，但目的是该API并非特定于Java。例如，存在Python绑定。

在XML一致性方面，有一个Oasis / NIST工作组，其工作是产生一组可接受的XML一致性测试和相关的基础结构。

在所有这些情况下，此过程都希望不抢先在此完成的工作，而是酌情进行协作以实现预期的结果。特别是，此过程将重点放在Java平台集成问题上，这不是任何现有工作的主要目标
```
