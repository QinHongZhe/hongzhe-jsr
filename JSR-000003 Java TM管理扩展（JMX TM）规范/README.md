# JSR-000003 Java TM管理扩展（JMX TM）规范


#### 规范描述
```
JMX TM规范将提供管理体系结构，API和服务，以构建基于Web的，分布式的，动态的和模块化的解决方案，以管理支持Java的资源。
```

#### 要求

```
-目标Java平台

    所有平台：完全Java，个人Java和嵌入式Java

    JMAPI将使向Java添加可管理性成为可能
    启用的设备，从网络电话和机顶盒到网络
    设备和服务器。

-解决了Java社区的需求

    Java社区需要通用的模块化Java
    解决以下问题的管理基础
    要求：

    +允许针对以下方面快速开发Java管理解决方案
      以下市场：消费者，企业计算，
      电信和数据通信。

    +提供可扩展的基于组件的架构
      从小型设备到大型电信交换机。

    +允许管理应用程序的可移植性
      支持多种硬件环境，协议，
      信息模型，数据库...

    +允许动态升级管理功能。

    +提供与旧式管理系统的集成。

-为什么不需要解决

    尚无通用的Java管理基础。这
    先前的JMAPI草案尚未达到规范阶段
    并且不能满足以下市场要求：

    +整合第三代新趋势
      基于Web的分布式管理。

    +利用最新的Java技术，例如JavaBeans
      和基尼。

    +与旧版或非Java环境（例如
      作为SNMP和CIM / WBEM）。

-待制定的规范

    JMAPI 2.0将提供管理架构，API和
    用于构建基于Web的，分布式的，动态的和
    模块化解决方案，用于管理Java支持的资源。

    JMAPI体系结构将适用于创建
    智能Java代理和管理应用程序，也可以是
    集成到旧版管理解决方案中。

    JMAPI 2.0将大力利用现有产品，并且
    因此应该在短期内导致成熟的规范
    以及商业实施的可用性。

-基础技术

    JMAPI规范将利用Java Dynamic
    管理套件技术。该产品提供了核心
    管理框架，它是Java管理的存储库
    豆子。基本上任何Java bean都可以在
    框架，然后公开管理功能。

    可以执行以下任一管理操作：

      +从本地，从同一VM，或
      +从远程VM，或者
      +来自远程非Java管理应用程序，或
        浏览器。

    该产品还提供了一组核心分布式
    可以部署在经理，代理或管理员中的管理服务
    代理/经理应用程序。

-建议的包装名称

    JMAPI API和服务将分为：

    +一套强制性服务，任何符合条件的
      实施将需要实施。

      我们建议将所有这些服务归类为
      以下软件包名称：

              javax.management.core

    +一组可选服务，任何符合条件的服务
      实施可能选择还是不实施。

    我们建议将每个服务定义为一个包
    以以下方案命名：

              javax.management。

    JMAPI测试套件将具有相同的模块化。
    声明符合JMAPI要求时，实现将
    必须提供可选服务的详尽清单
    他们支持，并将针对此“ JMAPI
    实施一致性声明”。

-可能的平台依赖性

    参考实现将依赖于
    JDK 1.2版本。

    JMAPI规范将引用现有的Java
    必要时使用JNDI，JDBC，JTS等规范。

-安全隐患

    没有。
    JMAPI将不包含安全功能。

-国际化的影响

    JMAPI API不处理信息的显示，但是
    而是处理信息的存储和检索。

    这些API依靠Java的内置功能进行操作
    与语言环境无关的数据，例如使用Unicode的字符串
    多字节字符集，因此将允许开发
    完全国际化的基于JMAPI的应用程序。

-本地化含义

    没有。

- 风险评估

    JMAPI不是现有Java平台的必需部分。
    因此，它不会危害现有平台，并且
    扩展或任何其他Java标准化项目。

    可交付成果的风险（规格，参考
    自实施以来，兼容性测试套件的数量很少）
    基础技术已经作为产品存在，
    相关的质量检查测试套件。

-现有规范已过时或过时

    JMAPI规范的此修订版将过时
    当前的JMAPI 0.8草案。

    没有基于此草案的商业产品
    今天在市场上。

-现有规格需要修订

    不适用。
```
